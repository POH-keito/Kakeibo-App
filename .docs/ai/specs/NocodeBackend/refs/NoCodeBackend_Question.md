## プラン・価格・Tier

### Tier の違いと機能

**Q: Tier 3-5 は「Pro」プランですか？主な違いは何ですか？**

**A:** はい、Tier 3-5 は Professional プランです。主な違いは以下の通りです：

- **Tier 1-2**: Starter プラン
- **Tier 3 以上**: Professional プラン
- **違い**: プロジェクト数、チームメンバー数、AI クレジット数が異なります

### 各 Tier の主な違い

**Q:** 各 Tier の主な違いを教えてください

**A:**

| Tier   | データベース数 | AI クレジット   | カスタムドメイン | チームメンバー | API 制限            |
| ------ | -------------- | --------------- | ---------------- | -------------- | ------------------- |
| Tier 1 | 3              | なし            | なし             | なし           | 20 リクエスト/10 秒 |
| Tier 2 | 15             | 15（1 回限り）  | なし             | なし           | 無制限              |
| Tier 3 | 50             | 50（1 回限り）  | 1 つ（無料）     | 最大 3 人      | 無制限              |
| Tier 4 | 100            | 100（1 回限り） | 1 つ（無料）     | 最大 10 人     | 無制限              |
| Tier 5 | 250            | 250（1 回限り） | 1 つ（無料）     | 最大 20 人     | 無制限              |

**Q: 追加のデータベースを購入できますか？**

**A:** はい、可能です。ダッシュボードの Billing セクションから追加購入できます。

- **価格**: 3 データベースパックで$49（1 回限り）
- **全 Tier 共通**: 価格は全 Tier で同じです

**Q: カスタムドメイン機能はどの Tier で利用できますか？**

**A:** Tier 3 以上で利用可能です。

- **無料提供**: Tier 3 以上には 1 つのカスタムドメインが無料で付属
- **追加購入**: 追加のカスタムドメインは$9/月のアドオンで購入可能
- **用途**: API ドキュメントを独自のドメインで公開（API 呼び出しもこのドメイン経由）

**Q: 無料プランと AppSumo プランの違いは？**

**A:**

- **無料プラン**: データベースあたり 1 テーブルのみ、AI 機能なし、SQL コマンドなし、API 呼び出し制限あり
- **AppSumo プラン**: 無制限のテーブルとカラム、AI 機能（Tier 2 以上）、SQL コマンド、API 呼び出し制限なし（Tier 2 以上）

## AI 機能・クレジット

### AI クレジットの仕組み

**Q: AI 機能は無制限ですか？それとも制限がありますか？**

**A:** 各 Tier で 1 回限りのクレジットが付与されます：

- **Tier 1**: AI 機能なし
- **Tier 2**: 15 AI クレジット
- **Tier 3**: 50 AI クレジット
- **Tier 4**: 100 AI クレジット
- **Tier 5**: 250 AI クレジット

**Q: 追加の AI クレジットの価格は？**

**A:** $49 で 100 AI エージェント呼び出しを購入できます。ダッシュボードの Billing セクションから追加購入可能です。

**Q: AI クレジットは月次で補充されますか？**

**A:** 現在は 1 回限りのクレジットです。将来的には定期補充への移行を検討中です。

**Q: BYOK（Bring Your Own Key）機能はありますか？**

**A:** 現在は未対応です。理由：

- Anthropic と OpenAI との 2 年間のバルク使用契約を締結済み
- 最近多くのユーザーが無制限 AI クレジットプランを購入済みのため、不公平になる可能性

将来的には Tier 2 以上で BYOK 機能の提供を検討中です。

**Q: AI エージェント呼び出しとは何ですか？**

**A:** AI エージェント呼び出しは、複数の LLM モデル（Claude 4、GPT 5 など）をバックエンドで使用し、以下の複数のアクションを 1 回の呼び出しで実行します：

- データベーススキーマの作成
- API エンドポイントの生成
- API ドキュメントの生成
- データベースの精緻化

1 回の呼び出しで完全なデータベースと API が生成されるため、非常に価値が高いです。

## データベース・ストレージ・制限

### データベースの仕様

**Q: データベースサイズの制限はありますか？**

**A:** データベースストレージ自体に制限はありませんが、レコード数に制限があります。

- **レコード制限**: テーブルあたり 100 万レコード（推奨）
- **注意**: 100 万レコードを超えても追加可能ですが、パフォーマンスが低下する可能性があります

**Q: テーブル数とカラム数の制限は？**

**A:** 無制限です。1 つのデータベースに無制限のテーブルとカラムを作成できます。

**Q: データベースエンジンは何を使用していますか？**

**A:** MySQL を使用しています。すべてのデータベースは当社のクラウドサーバーでホストされています。

**Q: 生のデータベース認証情報を取得できますか？**

**A:** いいえ、提供していません。データベースはすべて当社のクラウドサーバーでホスト・管理されており、API 経由でのみアクセス可能です。

**Q: データベースの場所はどこですか？**

**A:** 2 つのサーバーで管理されています：

- **Oracle Cloud**: 1 つのサーバー
- **AWS**: もう 1 つのサーバー（EU 地域：ドイツ、US 地域：アメリカ）

将来的にはシンガポールやインドへの展開も検討中です。

**Q: データベースをエクスポートできますか？**

**A:** はい、いつでもエクスポート可能です。

- **方法**: All databases → データベースの横の三点メニュー（⋮） → Export
- **形式**: SQL 形式でエクスポートされ、MySQL データベースと互換性があります

**Q: 既存の MySQL データをインポートできますか？**

**A:** データのインポートは現在サポートしていません。ただし、SQL コマンドを使用してテーブル構造をコピー&ペーストすることは可能です。

**Q: データベースを複製できますか？**

**A:** 現在は未対応ですが、実装予定です。

**Q: 削除したデータベース名を再利用できますか？**

**A:** はい、削除後は名前が解放され、再利用可能です。ただし、大文字小文字は区別されます（例：`KakeiboAppDB`と`kakeiboappdb`は異なる）。

## セキュリティ

### 認証とアクセス制御

**Q: シークレットキーのセキュリティはどうなっていますか？**

**A:**

- **推奨事項**: シークレットキーをクライアント側のコード（フロントエンドコード）に直接埋め込まないでください。環境変数として管理してください
- **Authorization ヘッダー**: シークレットキーは Authorization ヘッダーで Bearer トークンとして渡されます。これはクエリパラメータよりも安全で、暗号化されます
- **可視性**: ネットワーク呼び出しを確認しても、シークレットキーは呼び出し元のシステムからのみ表示されます

**Q: スコープ付きシークレットキーとは何ですか？**

**A:** 複数のキーを作成し、それぞれに権限スコープを設定できます：

- **Read**: データの読み取りのみ
- **Write**: 新規作成のみ
- **Update**: 更新のみ
- **Delete**: 削除のみ
- **All**: フルアクセス

**Q: ドメインホワイトリスト機能はありますか？**

**A:** はい、2025 年 11 月 3 日に実装されました。

- **機能**: 承認されたドメインからのみシークレットキーが機能するように制限
- **設定方法**: My databases → ⋯ → Secret keys → "Allowed domains"にドメインを入力
- **ワイルドカード対応**: `.example.com`のようなパターンで全サブドメインをカバー可能
- **多層防御**: スコープ付きキーと組み合わせて、より強力なセキュリティを実現

**Q: 行レベルセキュリティ（RLS）はサポートしていますか？**

**A:** MySQL はネイティブで RLS をサポートしていませんが、シークレットキーと WHERE 句を使用してセキュリティを提供しています。

**Q: 個別エンドポイントのセキュリティ設定は可能ですか？**

**A:** 現在は未対応ですが、要望として受け付けています。将来的には、各シークレットキーに対して特定のエンドポイントへのアクセス権限を設定できる機能を検討中です。

## 統合・連携

### フロントエンドツールとの統合

**Q: Greta、Lovable、Bolt.new などのツールと統合できますか？**

**A:** はい、可能です。NoCodeBackend は標準的な REST API エンドポイントを生成するため、HTTP リクエストを送信できるツールであれば統合可能です。

- **Swagger JSON**: API スキーマ全体を Swagger（OpenAPI）JSON 形式でエクスポート可能
- **統合方法**: Swagger JSON をコピーして、Greta、Cursor、Lovable などの AI コーディングツールに貼り付けるだけで、ツール側が API 仕様を完全に理解できます

**Q: WordPress と統合できますか？**

**A:** WordPress のデフォルトデータベースを置き換えることはできませんが、カスタムプロジェクトで API を呼び出すプラグインや方法があれば使用可能です。

**Q: Bubble.io、FlutterFlow、Webflow などと統合できますか？**

**A:** はい、可能です。これらのツールは HTTP リクエストを送信できるため、生成された API エンドポイントを使用して統合できます。

**Q: Zapier、Make.com、Activepieces などの自動化ツールと統合できますか？**

**A:** はい、可能です。これらのツールは HTTP リクエストを送信できるため、NoCodeBackend の API エンドポイントを呼び出してデータの追加、更新、削除、読み取りが可能です。

**Q: Supabase や Xano と比較してどうですか？**

**A:**

- **NoCodeBackend**: 主にノーコードユーザー向け。API 生成に特化し、迅速な開発を重視
- **Supabase**: 開発者向け。PostgreSQL データベースへの直接アクセスを提供
- **Xano**: より成熟したプラットフォームで多様な機能を提供。エンタープライズ向け

NoCodeBackend の特徴：

- AI によるスキーマとドキュメント生成
- 標準 REST エンドポイント
- Swagger ドキュメントの自動生成
- CORS サポート
- Bearer トークン認証

## 機能・制約

### API 機能

**Q: バルク操作（一括更新）はサポートしていますか？**

**A:** いいえ、現在はサポートしていません。Zapier や Pabbly Connect などの自動化ツールを使用して、Excel ファイルを読み取り、各行に対して API 呼び出しを実行することで実現可能です。

**Q: JOIN（テーブル結合）は可能ですか？**

**A:** はい、可能です。以下の 3 つの方法で作成できます：

1. **UI**: ビジュアルエディターを使用
2. **SQL**: SQL クエリを直接記述
3. **AI**: 自然言語で指示して AI が生成

**Q: フィルタリングや検索機能はありますか？**

**A:** はい、高度なクエリフィルターをサポートしています。

- **対応オペレーター**: GTE、LTE、GT、LT、NE、IN、LIKE
- **使用例**: `manufactured_date[gte]=2025-03-01&manufactured_date[lte]=2025-08-10`
- **部分一致**: `name[like]=asp`のように部分文字列検索が可能

**Q: ページネーションはサポートしていますか？**

**A:** JOIN 機能の実装後にページネーション機能の実装を予定しています。

**Q: ソート機能はありますか？**

**A:** はい、複数列でのソートが可能です。`sort`と`order`パラメータを使用します。

**Q: Webhooks は利用できますか？**

**A:** はい、2025 年 11 月 9 日に実装されました。

- **機能**: テーブルの変更（INSERT、UPDATE、DELETE）時に HTTP コールバックをトリガー
- **設定方法**: Developer settings → Webhooks → Instance、テーブル、イベント、ターゲット URL を設定

**Q: リアルタイム通信（WebSockets/SSE）はサポートしていますか？**

**A:** 現在は未対応です。Webhooks 機能が代替手段として利用可能です。

### データ型

**Q: サポートされているデータ型は何ですか？**

**A:**

- **基本型**: Text、Integer、Boolean、Float、Drop-down
- **特殊型**: Password（自動的に bcrypt でハッシュ化）
- **日付型**: Datetime
- **数値型**: Smallint、Bigint、Float、Double、Decimal

**Q: JSON 型や BLOB 型はサポートしていますか？**

**A:**

- **JSON 型**: 開発中です。近い将来に実装予定
- **BLOB 型**: 現在は未対応です。ファイルストレージサービス（S3、Dropbox など）を使用し、URL をテキストカラムに保存することを推奨します

**Q: ベクトルデータ型はサポートしていますか？**

**A:** 現在は未対応ですが、実装の可能性を検討中です。

### 計算機能

**Q: 計算カラム（自動計算フィールド）は作成できますか？**

**A:** はい、AI 機能を使用して作成できます。

- **例**: 「Products テーブルに`discounted_price`という計算カラムを作成し、`price`カラムの値の 50%を計算する」と自然言語で指示
- **機能**: 複雑な SQL やコードを書くことなく、AI が自動的に計算ロジックを実装

## 使用方法・ユースケース

### 基本的な使用方法

**Q: ユーザー認証システムを作成できますか？**

**A:** はい、可能です。

- **手順**:
  1. データベースを作成（例：「auth」）
  2. カラムを追加（`email`、`password`など）
  3. API が自動生成される
  4. フロントエンドの登録フォームとログインフォームを API に統合

**Q: パスワードリセット機能はありますか？**

**A:** はい、2025 年 9 月 28 日にネイティブサポートが追加されました。

- **機能**: SMTP 設定と API コマンドの自動生成
- **セキュリティ**: 自動生成されるリセットトークン（JWT）は 15 分で有効期限切れ
- **実装**: 認証 DB のセットアップ → SMTP 設定 → 生成された cURL コマンドを貼り付けるだけで完了

**Q: どのようなアプリケーションを構築できますか？**

**A:** 以下のようなアプリケーションを構築可能です：

- **CMS（コンテンツ管理システム）**: はい
- **LMS（学習管理システム）**: はい
- **E コマースストア**: はい
- **書籍リストアプリ**: はい
- **チャットアプリケーション**: はい
- **ユーザー管理システム**: はい

基本的に、データベースと API が必要なあらゆるアプリケーションに使用できます。

**Q: コードを書かずに使用できますか？**

**A:** はい、NoCodeBackend はノーコードツールです。技術的な知識がなくても、直感的なインターフェースでデータベースと API を作成できます。

## まとめ

NoCodeBackend は、コーディング不要でデータベースと API を迅速に構築できる強力なツールです。AppSumo の LTD プランでは、早期採用者に対して非常に寛大な制限と機能を提供しており、継続的な改善と新機能の追加が行われています。

**主な特徴**:

- 迅速な API 生成
- AI によるデータベース管理
- スコープ付きシークレットキーとドメインホワイトリストによるセキュリティ
- 幅広いツールとの統合
- 継続的な機能改善

**推奨事項**:

- シークレットキーは環境変数で管理
- 機密データには Password 型を使用
- 定期的にデータをエクスポートしてバックアップ
- サポートチームは迅速に応答し、フィードバックを積極的に取り入れている
